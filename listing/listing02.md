Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import (
	"fmt"
)


func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}


func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}


func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
Вывод программы:
2
1

В функции test, переменная x создана в блоке возврата, который не относится к локальной области видимости функции,
поэтому вызов отложенной функции (в блоке defer) изменит возвращаемое значение,
так как вызов будет обращен к области стека, в которой хранится возвращаемое значение.

В то время как в anotherTest переменная x создана в локальной области видимости функции.
Отложенная функция обновит значение x, но на результат это уже не повлияет,
так как возвращаемое значение к тому моменту уже будет записано в стек
и адрес локальной переменной не будет равен адресу области стека с возвращемым значением.
```
